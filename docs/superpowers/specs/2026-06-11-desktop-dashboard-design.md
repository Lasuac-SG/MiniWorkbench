# 桌面极客看板 (Desktop Geek Dashboard) - 系统设计说明书

## 1. 系统概述

本项目旨在打造一个常驻桌面的模块化极客数据看板，采用 C++20 与 Qt 6 (QML) 混合架构。系统融合了“灵动岛”的设计语言，具备边缘自适应吸附、悬停展开、动态网格微件布局等现代 UI 特性，并提供低延迟的本地 Git 仓库热力图统计及 TodoList 任务管理功能。

## 2. 核心交互设计 (UI/UX)

*   **平时状态 (信息胶囊)**：在屏幕边缘吸附时，以一个小巧的圆角矩形（胶囊）存在。可动态轮播或固定显示关键信息（如番茄钟倒计时、Todo 数量）。
*   **展开动效**：当鼠标悬停于胶囊上时，触发**自适应弹跳 (Bounce) 动画**。面板从胶囊所在位置向屏幕内部平滑展开，最大扩展至屏幕大小的 1/4。
*   **内部排版**：采用**网格布局 (Grid)**。微件（Widgets）在网格上自动对齐并能挤占空间；若微件内容过多或数量超出单屏限制，支持内部分页或滚动。
*   **交互模式**：
    *   **查看模式**：平时默认为查看模式，界面纯净，仅响应微件自身的业务交互（如点击任务、查看热力图详情）。
    *   **编辑模式**：长按面板或右键呼出菜单进入编辑模式。此模式下微件可拖拽（调整网格位置）及缩放（调整跨越的行/列数）。
*   **自由定位**：胶囊可被自由拖拽至屏幕的上下左右任意边缘，应用会自动计算正确的吸附位置并调整后续的展开方向。

## 3. 技术架构与选型

### 3.1 基础设施层 (C++ & OS 原生)
*   **动态窗口管理 (方案 B 实施)**：
    使用严格贴合内容的动态窗口。窗口初始大小等于信息胶囊尺寸。悬停展开时，C++ `WindowManager` 首先将不可见的系统窗口尺寸扩展至展开所需大小（维持背景透明），随后 QML 层执行视觉元素的平滑弹出动画；收起时反之。避免全屏透明遮罩带来的鼠标事件穿透等兼容性问题。
*   **屏幕吸附计算**：在 C++ 层实现逻辑，获取多显示器边界信息，根据拖拽结束时的坐标判断最近边缘，并动态设置 QML 界面的 `anchors` 及动画方向。

### 3.2 业务逻辑与数据存储层 (C++)
*   **数据库持久化**：使用 SQLite (`DatabaseManager` 单例)。存储应用配置（吸附位置、展开大小）以及各微件的状态（类型、`gridX`、`gridY`、`colSpan`、`rowSpan`）。
*   **混合刷新机制**：后台线程执行低频轮询（如每半小时一次）；窗口展开时即刻触发一次增量拉取。
*   **高性能 Git 解析后侧**：使用 `libgit2` 解析本地仓库的 commits 与增删行数。使用 C++ `QThread` 或 `QThreadPool` 将耗时任务放入子线程。完成后通过信号槽将数据传送回主线程的 QML Model。

### 3.3 视觉渲染层 (QML)
*   **网格坐标映射引擎**：设计自定义的 QML 组件将逻辑网格坐标 (`x, y, w, h` in units) 转换为像素级的真实布局坐标，并附带拖拽占位反馈及位置动画。
*   **高性能热力图渲染**：在 C++ 继承 `QQuickPaintedItem`，实现批量矩形绘制，直接抛给 QML 渲染，确保在大量 Git 提交数据下无渲染卡顿。

## 4. 模块结构与责任划分

1.  **WindowManager**：负责平台原生窗口属性控制、吸附边缘判定、实时调整透明窗口边界。
2.  **WidgetConfigModel**：基于 `QAbstractListModel`，为 QML 层提供当前存在的微件列表及坐标数据。
3.  **GitAnalyzer & HeatmapItem**：执行 Git 后台遍历，整理贡献权重，交给 `HeatmapItem` 直接渲染。
4.  **DatabaseManager**：全盘接管 SQLite 连接生命周期，负责初始化表结构并提供增删改查。
5.  **QML UI Components**：
    *   `MainCapsule.qml`：收起时的胶囊态。
    *   `ExpandedPanel.qml`：展开后的背景及基础框架。
    *   `GridLayoutEngine.qml`：负责编辑模式的拖拽逻辑、占位符显示以及网格重排。

## 5. 项目拓展性考量

*   由于微件数据结构抽象为类型 (`WidgetType`) 与网格参数，未来新增诸如番茄钟 (`PomodoroWidget`) 或天气微件等，只需增加对应的 QML 文件并在数据库插入新记录，无需大幅修改底层排版引擎。
