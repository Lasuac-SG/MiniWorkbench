桌面极客看板项目 - AI 编码代理执行指导书

致 AI 编码代理:
本文档为项目最高执行纲领。编写代码前必须严格遵守本文件的架构规范与核心要求。采取渐进式交付，每完成一个模块必须等待编译验证通过后再进行下一模块。

1. 前置环境依赖

C++ 标准： C++20。

构建系统： CMake 3.16及以上。

核心框架： Qt 6 包含 Core, Gui, Qml, Quick, Sql, Concurrent 模块。

第三方库： libgit2。通过 CMake 引入，禁止硬编码本地绝对路径。

目标平台： 跨平台。文件路径处理使用 QDir 或 std::filesystem，严禁硬编码分隔符。

2. 项目目标

开发常驻桌面的模块化数据看板。

核心形态： 启动即为无边框、背景透明窗口。默认隐藏在屏幕边缘，支持鼠标悬停 0.5 秒后平滑滑出。

UI 范式： 提供画布，允许通过配置动态加载、拖拽、缩放多个组件。

业务领域： 提供 TodoList 任务管理、基于本地 Git 仓库轮询的代码贡献热力图。

3. 核心开发要求

3.1 强制设计原则与规范

抽象： 精确定义接口边界，对外仅暴露必要交互契约。

DRY： 严禁代码复制粘贴，重复逻辑必须提取。

KISS： 拒绝过度设计，选择最简单直接的实现路径。

YAGNI： 仅在需要时添加代码，严禁编写冗余抽象层。

LoD： 严格遵守交互限制，对象只与其直接依赖项通信，禁止链式调用暴露内部结构。

高内聚低耦合： 相同职责高度集中；UI 渲染层与业务数据逻辑层必须物理隔离与单向依赖。

代码精简： 拒绝冗余设计，善用 C++20 语法糖。

注释详细： 类声明与复杂算法逻辑必须包含 Doxygen 风格注释说明时间复杂度和内存所有权。

面向对象与封装化： 严格控制访问权限。禁止暴露不必要内部指针，C API 必须完全封装在 C++ 类中。

3.2 核心规范约束

MVC架构： QML 仅负责渲染和动画。业务逻辑、文件读写、Git 轮询必须在 C++ 中执行。C++ 与 QML 的列表数据传递强制使用继承自 QAbstractListModel 的方式。

防御性编程与 RAII 资源管理： 禁止使用裸指针分配内存。强制使用 std::unique_ptr 或 QScopedPointer。封装 libgit2 时必须使用自定义 Deleter 的智能指针接管资源释放。

线程安全与熔断： Git 历史遍历必须在独立子线程中执行。向主线程传递数据必须使用 Qt 信号与槽。

4. 项目功能细节与执行拆解

执行指令： 按照以下 Block 顺序逐一实现。每完成一个 Block，提供 CMakeLists.txt 更新和源代码，提示编译测试。

Block 1: 基础设施与窗口管理层

任务： 搭建 CMake 工程。创建无边框、背景透明的 Qt Quick 主窗口。

细节： 实现 WindowManager 类注册到 QML。实现 QML 边缘侦测逻辑：悬停屏幕边缘 0.5s 触发属性动画滑出，移开滑回。

Block 2: 动态微件系统基础

任务： 实现可拖拽、可缩放 UI 框架。

细节： C++ 建立 WidgetConfigModel 存储微件坐标、尺寸、类型。QML 读取模型。内部封装 DragHandler 实现拖拽，边缘封装 MouseArea 实现缩放，结束后将新坐标回写至 C++。

Block 3: SQLite 本地存储持久化

任务： 引入 QtSql 模块，持久化微件状态和基础设置。

细节： 实现 DatabaseManager 单例。启动时自动建表，提供基础增删改查 API，确保重启状态不丢失。

Block 4: 高性能 Git 解析后端

任务： 封装 libgit2，提取代码增删行数。

细节： 实现 GitAnalyzer 类。实现基于后缀名规则的过滤逻辑。解析指定日期内的变更。必须在子线程执行，完成后通过信号通知主线程。

Block 5: 热力图渲染层

任务： 接收 Git 解析数据，利用 CPU或GPU 高效绘图。

细节： 创建继承自 QQuickPaintedItem 的 HeatmapItem 类并注册到 QML。暴露数据接收接口。在 paint 函数中根据微件尺寸动态计算绘制参数批量绘制矩形，实现无损重绘。

Block 6: TodoList 微件与最终联动

任务： 实现任务管理微件，纳入热力图权重。

细节： 开发 TodoList 的 C++ Model 和 QML 视图。定义统一贡献值接口，融合数据源统一输入给 HeatmapItem 渲染。
